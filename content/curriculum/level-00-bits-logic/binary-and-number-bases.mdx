Last lesson gave you a single bit: one yes/no answer, not worth much by itself. But we already hinted at the payoff — line enough bits up side by side and their combinations multiply fast. This lesson turns that stack into something you can actually count with: real numbers, written in base two, plus the shorthand (hexadecimal) engineers use so they don't have to stare at forty digits of 1s and 0s at once.

The reassuring secret is that binary isn't a different kind of math. It's the exact same place-value trick you learned for decimal in grade school — just with fewer digits to work with per column. Once that clicks, converting between bases stops being a memorized ritual and becomes something you can rebuild from scratch any time you need it.

## Place value is the whole trick

Take the decimal number 372. You don't think of it as three random symbols — you automatically read it as `3×100 + 7×10 + 2×1`. Each position is worth ten times the position to its right, because we're counting in base ten. That's <b>positional notation</b>: a digit's value depends on where it sits, not just what symbol it is.

Base ten isn't mathematically special — it's an accident of having ten fingers. The exact same rule works for any base you choose: pick a base `b`, use digits from 0 up to b−1, and each position is worth b times the one before it. Binary is just this rule with b = 2, so the only digits available are 0 and 1.

## Counting in binary

In binary, the place values aren't 100s and 10s — they're powers of two: 8, 4, 2, 1 for a 4-bit number. To read `1011`, add up the place values where there's a 1: 8 + 0 + 2 + 1 = 11. That's the entire algorithm — no lookup table required, just the same column-by-column sum you'd do for a decimal number.

Counting upward in binary rolls over exactly like a decimal odometer, just sooner: 00, 01, 10, 11 exhausts every 2-bit pattern in four steps instead of ten. Once you can watch that rollover happen — 0111 becoming 1000, four columns flipping at once — binary addition stops feeling foreign.

## Hexadecimal: binary's shorthand

Writing out a 32-bit binary number by hand is unwieldy and easy to miscount. <b>Hexadecimal</b> (base 16) exists purely as a human convenience: since 16 is exactly 2⁴, every possible pattern of 4 bits maps onto exactly one hex digit (0–9, then A–F for 10–15). No remainder, no awkward overlap — it's the cleanest possible shorthand for binary.

To convert, just chop the binary number into groups of four bits, starting from the right, and translate each group independently. You never need to think about the whole number's value at once.
