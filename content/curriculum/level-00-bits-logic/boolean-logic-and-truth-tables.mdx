You can now read and write numbers in binary. But computers don't just store numbers — they make decisions: compare two values, check a condition, decide which instruction to run next. That requires a formal system for combining true/false values, and it already existed a century before the first computer: <b>Boolean logic</b>, named for the 19th-century mathematician George Boole, who first showed that logic itself could be manipulated like algebra.

By the end of this lesson you'll know the three core operators — AND, OR, NOT — plus XOR, be able to build any of their truth tables from memory, and see exactly how "true and false" is just another name for the "1 and 0" bit you met in lesson one. Logic, numbers, and bits are about to turn out to be the same idea wearing different clothes.

## True and false are just another pair of states

A boolean value only ever has two possibilities: true or false. Sound familiar? That's exactly the shape of a bit. In real hardware, a "true" wire and a "1" wire are the same physical thing — a boolean variable is just a bit that we've agreed to talk about using the words true and false instead of 1 and 0.

George Boole's insight in 1854 was that you could treat these two values algebraically: instead of adding and multiplying numbers, you combine truth values with logical operators. That algebra — Boolean algebra — turned out, a century later, to be exactly the mathematics needed to describe electronic circuits. Boole had no idea he was designing the theory behind every computer ever built.

## AND, OR, and NOT

`AND` is a strict gatekeeper: the result is true only when <i>every</i> input is true. "It's raining AND it's cold" is only true on days that are both — one condition failing sinks the whole expression.

`OR` is more permissive: the result is true if <i>at least one</i> input is true (including, importantly, when both are). `NOT` is the simplest of all — it takes a single input and flips it, true to false or false to true.

## XOR: exactly one, not both

`XOR` ("exclusive or") looks like OR at first glance, but differs at exactly one row: when both inputs are true, OR says true, XOR says false. XOR is true only when the two inputs <i>differ</i> — one true and one false, in either order.

That single-row difference isn't a technicality — XOR turns out to be exactly the operation you need to add two bits together and to flip individual bits without touching the rest, which is why it reappears constantly once we start building circuits in the next lesson.
