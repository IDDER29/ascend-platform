Pointers feel like dark magic until one idea clicks: **a pointer is just a
variable that holds an address.** That's the entire secret. Let's make it
real — and you can run the code as you go.

## Everything lives at an address

Every variable your program makes lives *somewhere* in memory, and every
somewhere has a numeric address — its house number. Writing `int x = 42;`
picks an address and stores 42 there. The `&` operator hands you that
address.

## Follow the address with *

The `*` operator means "go to that address and use what's there." So `*p`
**reads** x — and assigning to `*p` **writes** x. Don't take our word for
it:
