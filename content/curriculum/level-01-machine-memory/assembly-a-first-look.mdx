Last lesson you met the ALU, the registers, and the control unit that ties them together. Now you get to watch them work, one line at a time. <b>Assembly language</b> is the closest you can get to reading a CPU's mind in something resembling English: one line of assembly is (almost always) one instruction the control unit will fetch and execute.

You won't write much assembly on Ascend — Level 02 hands you C instead, a language that lets a compiler generate this stuff for you. But reading a little assembly now means that, for the rest of your CS life, "the compiler" and "the debugger" stop being black boxes. Let's look inside one.

## From binary opcodes to mnemonics

Every instruction a CPU executes is, underneath, a pattern of bits — an <b>opcode</b> that says which operation to do, plus bits naming which registers or values it touches. Nobody sane reads or writes that directly. <b>Assembly language</b> gives each opcode a short, memorable <b>mnemonic</b> — `mov`, `add`, `cmp` — and lets you name registers instead of memorizing bit positions.

An <b>assembler</b> turns mnemonics back into the exact bit patterns the CPU expects; a <b>disassembler</b> goes the other way, turning a compiled program's bytes back into readable mnemonics. Assembly and machine code describe exactly the same instructions — assembly is just the version a human can read without going insane.

## Anatomy of an instruction

Almost every instruction has the same shape: an <b>operation</b> followed by one or more <b>operands</b>. Operands can be a register (`eax`), an immediate constant (`5`), or a memory address. A handful of mnemonics cover most of what you'll ever read: `mov` copies a value, `add`/`sub` do arithmetic, `cmp` compares two values by quietly subtracting them and keeping only the flags, and a family of `j*` instructions (jump-if) move the program counter somewhere else <i>if</i> a flag says to.

That last family is how loops and if-statements are actually built at the hardware level — there is no if-statement in machine code, only "compare, then maybe jump."

## Reading a small function

Here's a tiny function that returns the larger of two numbers — the kind of thing you'll write in about one line of C in a few weeks. Read it the way you'd read the fetch-execute loop: one instruction at a time, tracking what's in each register.

`cmp` sets the flags by comparing the two arguments; `jle` reads those flags and skips the next line only when the first argument is less than or equal to the second. No magic — just registers, an ALU comparison, and a conditional jump, exactly as promised in Lesson 1.

<CodeBlock lang='asm' linesJson='["max:", "    mov  eax, edi     ; assume a (edi) is the answer", "    cmp  esi, eax     ; compare b (esi) to eax", "    jle  .done        ; if b &lt;= a, keep eax as-is", "    mov  eax, esi     ; otherwise, the answer is b", ".done:", "    ret               ; return the value now in eax"]' />
